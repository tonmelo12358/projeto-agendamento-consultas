# Gestão de Riscos — App de Agendamento de Consultas Médicas

## Descrição do projeto

Este repositório reúne os artefatos de um exercício de gerenciamento de riscos aplicado a um cenário fictício: o desenvolvimento de um aplicativo móvel para agendamento de consultas médicas, com funcionalidades de cadastro de usuários, gestão de agenda médica, agendamento de consultas, notificações e integração com um sistema externo de prontuário.

O projeto está em fase intermediária de desenvolvimento, conduzido por uma equipe de 4 desenvolvedores e 1 tester, e enfrenta desafios reais de gestão: instabilidade na integração com o sistema externo, mudanças de requisitos solicitadas por stakeholders e sobrecarga da equipe frente aos prazos estimados.

## Objetivo do exercício

Aplicar, com apoio de um LLM, as etapas centrais do processo de gerenciamento de riscos em projetos: identificação, análise qualitativa, definição de estratégias de resposta e comunicação executiva do status do projeto a stakeholders não técnicos, culminando na organização desses artefatos em um repositório versionado no Git/GitHub.

## Organização do repositório

```
projeto-agendamento-consultas/
├── README.md                          # este arquivo
├── riscos/
│   ├── identificacao.md                # Etapa 1 — lista de riscos, descrição e contexto
│   ├── analise.md                      # Etapa 2 — análise qualitativa (probabilidade x impacto) e fatores condicionantes
│   └── respostas.md                    # Etapa 3 — estratégias de resposta (evitar/mitigar/transferir/aceitar)
└── comunicacao/
    └── status-stakeholders.md          # Etapa 4 — comunicação de status para stakeholders
```

O histórico de commits deste repositório reflete a evolução da atividade: criação da estrutura inicial, seguida pela identificação de riscos, análise qualitativa, definição de estratégias de resposta e, por fim, a comunicação para stakeholders (Etapas 1 a 4 do exercício).
